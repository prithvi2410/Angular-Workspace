Change the input and check for ngOnChanges

ngOnChanges is called for primitive data types but when we pass complex objects as input
ngOnChanges is called only if the refrence is changed.
Here if you change String input ngOnChanges is called but if you change Object.Name input is changed ngOnChanges is not called.
it is because the refrence is not change only its property is changed.
